# Arabic News Text Classification - Project Structure

## Overview
This project has been refactored to separate frontend (HTML, CSS, JS) from the backend Flask application, following modern web development best practices.

## Directory Structure

```
News-Text-Classification/
├── backEnd.py                      # Flask application (main backend)
├── train_model.py                  # Script to train and save the ML model
├── model.pkl                       # Trained model (generated by train_model.py)
├── requirements.txt                # Python dependencies
├── arabic_news.csv                 # Training dataset
├── README.md                       # Original project README
├── PROJECT_STRUCTURE.md            # This file
│
├── templates/
│   └── index.html                  # Main HTML template (Jinja2 format)
│
├── static/
│   ├── css/
│   │   └── style.css               # All CSS styles (extracted from HTML)
│   ├── js/
│   │   └── script.js               # All JavaScript logic (extracted from HTML)
│   └── images/
│       ├── confusion_matrix.png    # Model evaluation visualization
│       ├── eda_plots.png           # Exploratory data analysis plots
│       └── model_comparison.png    # Model performance comparison
│
├── notebooks/
│   ├── arabic_news_classification.ipynb  # Original Jupyter notebook
│   └── sample.ipynb                      # Sample notebook
│
└── WebPage_original.html.bak       # Original monolithic HTML file (backup)
```

## Key Changes

### 1. **Frontend Separation**
   - **HTML**: `templates/index.html` - Clean, semantic HTML using Jinja2 templating
   - **CSS**: `static/css/style.css` - All styles extracted into a separate file
   - **JavaScript**: `static/js/script.js` - All client-side logic extracted

### 2. **Backend Integration**
   - **Flask App**: `backEnd.py` - Serves the frontend and provides API endpoints
   - **Routes**:
     - `GET /` - Serves the main HTML template
     - `POST /predict` - Accepts JSON with Arabic text, returns classification

### 3. **Model Pipeline**
   - **Training**: `train_model.py` - Trains the model and saves it as `model.pkl`
   - **Preprocessing**: Includes Arabic-specific text preprocessing (diacritics removal, normalization, stemming)
   - **Features**: TF-IDF vectorization with unigrams and bigrams
   - **Classifier**: Logistic Regression (achieves ~92% accuracy)

## Installation & Setup

### 1. Install Dependencies
```bash
pip install -r requirements.txt
```

### 2. Train the Model (if model.pkl doesn't exist)
```bash
python train_model.py
```

### 3. Run the Flask Application
```bash
python backEnd.py
```

The application will be available at `http://localhost:5000`

## API Documentation

**Request:**
```json
{
  "text": "فاز المنتخب الوطني بالمباراة"
}
```

**Response:**
```json
{
  "category": "رياضة (Sports)"
}
```

**Categories:**
- `رياضة (Sports)`
- `سياسة (Politics)`
- `اقتصاد (Economy)`

## Development Notes

### Adding New Features
1. **New CSS**: Add to `static/css/style.css`
2. **New JS**: Add to `static/js/script.js`
3. **New HTML**: Modify `templates/index.html`
4. **New Routes**: Add to `backEnd.py`

### Debugging
- Flask debug mode is enabled by default in `backEnd.py`
- Check browser console for JavaScript errors
- Use `curl` or Postman to test API endpoints directly
